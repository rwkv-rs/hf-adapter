# A8/W8 speed-policy acceptance (RTX 4090)

Date: 2026-07-11

## Configuration

- Model: RWKV-7 G1D 0.4B HF
- GPU: NVIDIA GeForce RTX 4090 (SM89)
- dtype: fp16
- backend: native graph, fused recurrent attention
- prompt: 32 tokens
- measured decode: 512 tokens
- policy: `speed`; dynamic A8/per-channel W8 is applied to `lm_head`

This policy deliberately preserves dense small/body projections. Full-memory
W8 is a separate unfinished lane; the purpose here is a deployable W8 mode that
reduces memory without slowing any measured decode batch.

## Result

| batch | W8/fp16 speed | W8 tok/s | prompt cosine | final cosine | next token |
|---:|---:|---:|---:|---:|---:|
| 1 | **1.0316x** | 476.8 | 0.999994 | 0.999996 | match |
| 2 | **1.0166x** | 869.0 | 0.999984 | 0.999991 | match |
| 4 | **1.0156x** | 1578.9* | >=0.999983 | >=0.999981 | match |
| 8 | **1.0171x** | 2244.8 | 0.999976 | 0.999961 | match |

`*` Batch 4 is the median of three confirmations (`1.0156x`, `1.0205x`,
`1.0135x`).

Model payload falls from `859.8 MB` to `796.0 MB` (`0.9258x`). The modest
reduction is intentional: this is the speed policy, not the maximum-memory W8
policy. The graph path uses direct output-buffer writes to remove an avoidable
copy and a one-row W8A16 GEMV route to avoid padding batch 1 to tensor-core
minimum rows.

## Reproduce

```bash
python bench/bench_native_quant_e2e_decode.py \
  --hf-dir /path/to/rwkv7-g1d-0.4b-hf \
  --dtype fp16 --device cuda --attn-mode fused_recurrent \
  --fast-token-backend native_graph \
  --quantizations none a8w8 --policy speed --min-params 1 \
  --batch-size 1 --prompt-tokens 32 --decode-tokens 512 --warmup 1
```

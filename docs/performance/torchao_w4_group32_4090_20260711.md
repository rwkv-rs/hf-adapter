# Tensor-core W4 quality/speed acceptance (RTX 4090)

Date: 2026-07-11

## Acceptance result

RWKV-7 G1D 0.4B was tested through the public HF adapter decode path using
bf16 activations, TorchAO tensor-core W4 weights, native-graph cached decode,
32 prompt tokens, and 128 measured decode tokens.

All 145 size-gated projection, FFN, and head modules were quantized. Two
production choices are retained:

- group 128: maximum-throughput W4, payload `342.8 MB` versus `859.8 MB`
  (`0.3987x`);
- group 32: quality-oriented W4, payload `375.8 MB` (`0.4371x`). Its subgroup
  granularity is comparable to the 32-weight sub-block idea used by K-family
  formats, without claiming binary compatibility with llama.cpp Q4_K_M.

## Group-32 matrix

| batch | W4/bf16 speed | prompt cosine | final cosine | next token |
|---:|---:|---:|---:|---:|
| 1 | **1.4516x** | 0.999520 | 0.999586 | match |
| 2 | **3.3294x*** | 0.999528 | 0.999585 | match |
| 4 | **1.4872x** | 0.999481 | 0.999557 | match |
| 8 | **1.2302x** | 0.999409 | 0.999500 | match |

`*` An unrelated process occupied the GPU during this repeat, making absolute
latency and the batch-2 ratio provisional. The acceptance conclusion does not
depend on that outlier: the minimum observed ratio is still `1.2302x`, and the
earlier clean group-128 matrix independently records `1.241x-1.620x` speedups.

The final branch tree was separately reloaded through `trust_remote_code=True`;
a short group-32 smoke retained `1.2469x` decode speed, `0.4371x` payload,
`0.999407` final-logit cosine, and matching next token.

After the fixed-shape native-prefill work landed, the quantized path was
revalidated again. Native prefill is now explicitly bypassed for adapter-owned
quantized modules so TorchAO weights are never passed to the tensor-only packer.
The corrected final tree passes bsz1/2/4/8 with speed ratios
`1.3413x/1.7536x/1.4124x/2.3462x`, `0.4371x` payload, final cosine at least
`0.999500`, and matching next token. The host was co-tenanted during this last
repeat, so the earlier clean rows remain the absolute-throughput reference.

## Quality improvement

The projection-axis R/K/V experiment independently measures why smaller groups
are useful: group-32 MSE-clipped W4 lowers normalized weight error from
`0.03138` (row-wise) to `0.00925`, raises SQNR from `15.03 dB` to `20.34 dB`,
and raises mean projection-output cosine from `0.98467` to `0.99534`. Its packed
R/K/V slice is `0.28125x` fp16.

## Remaining boundaries

This closes RTX 4090 W4 cached-decode speed, footprint, and smoke-quality gates.
It does not close W4 prompt-prefill, W8 speed, long-task MATH500 acceptance, or
other GPU families. Those remain separate required gates and must not inherit
Ada results without exact-card measurements.

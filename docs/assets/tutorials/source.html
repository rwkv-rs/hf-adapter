1

$ python -m venv .venv $ python -m pip install -e .
2

Check the machine

$ python examples/check_environment.py RESULT: READY
3

Start with the public 0.4B checkpoint and official vocabulary.

hf download BlinkDL/rwkv7-g1 ... convert_rwkv7_to_hf.py ...

[PASS] Model directory generate.py --max-new-tokens 8

Use the native backend for the first compatibility run. Optional FLA optimization can be added later on supported Linux NVIDIA systems.
RWKV-7 HF / speculative decoding

Draft quickly, verify exactly, correct when needed

INFERENCE

1. Draft model

Proposes a short block of tokens using recurrent cache.

draft_tokens = 4 propose: A B C D
draft_forward_calls > 0
→

2. Target model

Verifies the proposal. Accepted tokens are kept; a mismatch is corrected.

target.verify(A B C D) accept / correct / resync
target_forward_calls > 0
→

3. Exact output

Greedy output must equal normal target generation.

speculative_stats {...} decoded ... PASS
Correctness first: same-model draft proves the API contract, not speed. A smaller compatible draft is required for a real speed experiment.
Command: python tests/test_speculative_decode.py --model MODEL --max-new-tokens 8 --draft-tokens 4
RWKV-7 HF / single-GPU training

Prove gradients before starting a real fine-tune

TRAINING
1

Install training extras

$ python -m pip install -e ".[train]"

Includes PEFT, Accelerate, datasets, TRL, and DeepSpeed.

2

LoRA backward smoke

test_peft_lora.py --model MODEL nonzero_grad_count > 0

Checks trainable parameters, finite loss, backward, and non-zero gradients.

3

Trainer smoke

test_native_trainer_smoke.py \ --max-steps 2 --batch-size 2 NATIVE TRAINER PASS

Checks a short optimizer update through the HF Trainer path.

Scope: a smoke pass proves compatibility, not convergence or dataset quality. A production run still needs a real dataset, evaluation split, checkpoints, and loss monitoring.
Disable cache during training. Start with 0.1B or 0.4B and confirm memory before increasing sequence length or batch size.

DeepSpeed ZeRO-2 and ZeRO-3 smoke workflow

Replicates parameters, partitions optimizer states and gradients.

GPU 0

opt/grad shard
Lower optimizer + gradient memory

ZeRO-3

Also partitions model parameters across workers.

GPU 0
param shard
GPU 1
param shard
Lowest per-GPU parameter memory
$ NPROC_PER_NODE=2 ZERO_STAGE=both MODEL=MODEL \ bash scripts/run_zero_training_smoke.sh PASS + rows written to bench/results.jsonl
Run on Linux/WSL2 with two visible CUDA GPUs. A one-step smoke does not prove long-run convergence or optimizer/scheduler continuity.
ZeRO is distributed training state sharding. Do not describe these rows as tensor-parallel inference evidence.

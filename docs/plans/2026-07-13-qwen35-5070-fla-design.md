# Qwen3.5 RTX 5070 FLA Benchmark Design

> **Historical design, implemented.** The initial core-FLA/Torch-conv rows
> described below were an intermediate step. The promoted 5070 result uses
> full FLA plus Triton causal convolution and is recorded in
> [`../../bench/5070_qwen35_full_fla_bsz8_20260714/README.md`](../../bench/5070_qwen35_full_fla_bsz8_20260714/README.md).
> Statements such as “current” and “open” below are scoped to 2026-07-13.

## Goal

Measure the official Qwen3.5 reference with its optimized FLA Gated DeltaNet
path on the exact `NVIDIA GeForce RTX 5070 Laptop GPU` (`sm_120`). The earlier
V100 forced-Torch matrix remains diagnostic evidence and must not be merged
into this result set.

## Operator contract

Package presence alone is not evidence of acceleration. Before timing, the
worker inspects every Qwen3.5 Gated DeltaNet layer and requires:

- FLA `chunk_gated_delta_rule` for prompt prefill;
- FLA `fused_recurrent_gated_delta_rule` for cached decode;
- FLA `FusedRMSNormGated` for gated normalization.

These operators form `qwen_fla_core_contract_pass`. The optional
`causal-conv1d` prefill/update operators are tracked under a separate contract.
The current Windows 5070 environment can execute the FLA core but does not have
the CUDA `causal-conv1d` extension, so accepted rows are labeled
`qwen_fla_gated_delta_rule_torch_conv`. They are optimized FLA-core rows, not
full-fusion claims. `--qwen-backend torch` remains a diagnostic oracle lane.

## Exact-card route

The initial environment is Windows, RTX 5070 Laptop `sm_120`, PyTorch
`2.11.0+cu128`, Transformers `5.12.1`, FLA `0.5.1`, and bitsandbytes `0.49.2`.
All telemetry records the exact GPU name, compute capability, driver-visible
CUDA stack, operator origins, and dependency versions. Blackwell behavior must
not be generalized to V100, Ampere, Ada, or other 50-series cards.

## Matrix and acceptance

The first complete matrix is RWKV-7 1.5B versus Qwen3.5 2B with prompt
128/512/2048, decode 128/512, bsz 1/2/4/8, and fp16/bnb8/bnb4: 144 raw rows
forming 72 comparison cells. The one-command runner is
`bench/run_5070_qwen35_fla_matrix.ps1`; results are resumable and live under
`D:\bench\qwen35-5070-fla-20260713` before a reviewed artifact is copied into
the repository.

Before the matrix, the runner executes separate FLA and forced-Torch
prompt128/decode8/bsz1 smokes. FLA rows must report the exact 5070 identity,
`qwen_fla_core_contract_pass=true`, and an FLA effective backend. Throughput
promotion additionally requires the 72-cell matrix and explicit FLA-vs-Torch
logits/greedy alignment. Larger Qwen3.5 4B/9B rows are follow-up feasibility
lanes on the 8GB card; OOM rows are evidence, not performance passes.

## Result

The 2026-07-13 exact-card run completed 144/144 passing raw rows and 72/72
joined cells. All Qwen rows pass the FLA core contract and use the explicit
`torch_conv` effective-backend label. The FLA/Torch probe matches 8/8 greedy
tokens with prompt/final cosine `0.99999076` / `0.99999213`.

The strict speed gate remains open: 35/72 cells pass both 1.05x thresholds.
Decode is >=1.0x in 72/72 and >=1.05x in 71/72, while prefill is >=1.0x in
41/72 and >=1.05x in 35/72. Footprint and peak VRAM are lower in all 72 cells.
Canonical evidence is `bench/5070_qwen35_fla_matrix_20260713/`.

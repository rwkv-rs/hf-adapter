# RTX 5090 Blackwell native MM8/MM4 fresh-process quant matrix (2026-07-05)

This artifact is the formal RTX 5090 / Blackwell native quantization matrix for
RWKV-7 HF adapter `native_mm_policy=speed`.  Each fp16/mm8/mm4 row was run in a
fresh Python subprocess by `bench/run_blackwell_quant_matrix.py`; fp16 rows save
baseline logits/tokps and quantized rows reload the matching baseline.  This is
important for 7B-class rows on a 32GB RTX 5090 because consecutive in-process
large-model loads can fragment peak VRAM.

Raw data:

- `results_5090_blackwell_quant_matrix_full.jsonl`: 216 pass rows.
- `summary.md`: full Markdown table plus ratio summary generated by
  `bench/summarize_blackwell_quant_matrix.py`.

Matrix:

- GPU: NVIDIA GeForce RTX 5090, 32GB class.
- Models: 1.5B, 2.9B, 7.2B converted HF directories.
- Quantization: fp16 baseline, native MM8 speed policy, native MM4 speed policy.
- Prompt tokens: 128, 512, 2048.
- Decode tokens: 128, 512.
- Batch sizes: 1, 2, 4, 8.
- Native policy: `speed` (`lm_head` only), `min_params=8_000_000`.
- Backend: FLA-free native model path with cached native JIT decode telemetry.

Interpretation:

- Correctness is stable across the matrix: every MM8/MM4 row matches the fp16
  greedy next token, and logits cosine remains high.
- Footprint decreases on every quantized row.  Because this is the speed policy
  (`lm_head` only), footprint reduction is intentionally modest compared with
  the `memory` policy.
- Many Blackwell rows are fp16-or-better, especially batch-size-8 medium/long
  decode rows.  The median is near fp16 for 1.5B/2.9B and below fp16 for 7.2B.
- 7.2B large-pressure rows, especially `bsz=8`, `prompt=2048`, `decode=512`,
  expose the remaining kernel/dispatch gap; they are correctness-passing but
  not speed-passing.

13.3B boundary note:

- The 13.3B official checkpoint was pulled from ModelScope LFS on the same
  server (`rwkv7-g1g-13.3b-20260523-ctx8192.pth`, ~25GB on disk; LFS pointer
  size says 26,540,868,485 bytes).
- The current converter attempted to build a full HF template while also holding
  the official checkpoint in CPU memory. On the 48GB RAM / no-large-swap 5090
  rental, that conversion did not produce a model directory. No 13.3B fp16/MM8/
  MM4 speed row is claimed in this artifact.
- The next 13.3B step is a low-memory/streaming converter or a larger-RAM host,
  then the same fresh-process boundary probe can be reused.
